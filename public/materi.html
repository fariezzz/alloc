Materi Memory Allocation

1. Pengertian dan Tujuan

Manajemen Memori adalah komponen sistem operasi yang memastikan sumber daya memori utama digunakan secara efisien agar CPU selalu memiliki proses yang siap dijalankan.

  • Allocation & Deallocation: mengatur pemakaian dan pelepasan memori proses.
  • Protection: mencegah proses saling mengakses memori.
  • Mapping: menerjemahkan alamat logis ke fisik.
  • Swapping: menukar proses antara RAM dan disk.

2. Jenis Memory Allocation

Memory allocation dibagi menjadi dua jenis:

a. Static (Fixed Partitioning)

Memori dibagi tetap sejak awal. Tiap partisi hanya menampung satu proses.

  • Kelebihan: mudah diatur.
  • Kekurangan: internal fragmentation.
b. Dynamic (Variable Partitioning)

Ukuran partisi menyesuaikan kebutuhan proses.

  • Kelebihan: lebih fleksibel.
  • Kekurangan: external fragmentation.

3. Algoritma Contiguous Memory Allocation

  • First Fit: blok kosong pertama yang cukup besar.
  • Best Fit: blok terkecil yang masih muat.
  • Worst Fit: blok terbesar yang tersedia.

4. Fragmentasi Memori

Jenis Penjelasan Penyebab
Internal Blok memori lebih besar dari kebutuhan proses. Fixed Partitioning
External Ruang kosong terpisah-pisah di antara blok terisi. Variable Partitioning

5. Contoh Kasus

Partisi: 100, 500, 200, 300, 600 KB.
Proses: 212, 417, 112, 426 KB.

Tentukan hasil untuk algoritma First-Fit, Best-Fit, dan Worst-Fit.

6. Kesimpulan

  • Manajemen memori penting untuk efisiensi dan keamanan proses.
  • Fragmentasi adalah masalah utama dalam alokasi berurutan.
  • Paging dan segmentation adalah solusi modern.
